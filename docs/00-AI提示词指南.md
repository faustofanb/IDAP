## 🎯 项目概览

### 项目定位

IDAP 是一个**企业级智能数据分析平台**，核心价值是让业务人员通过自然语言与数据对话，无需编写 SQL 即可获取数据洞察。

### 核心功能

| 功能模块         | 描述                | 技术实现                      |
| ---------------- | ------------------- | ----------------------------- |
| **NL2SQL**       | 自然语言转 SQL 查询 | LangChain + OpenAI/Anthropic  |
| **智能可视化**   | 自动生成数据图表    | ECharts + 规则引擎            |
| **RAG 知识检索** | 企业知识库问答      | FAISS/Milvus + Embedding      |
| **AI 洞察**      | 自动分析与建议      | LLM 推理 + Prompt Engineering |

### 项目目标

- **性能**: NL2SQL 生成 ≤3s，端到端响应 ≤8s
- **准确率**: SQL 生成准确率 ≥75%
- **并发**: 支持 100+ 并发用户
- **安全**: SQL 注入防护，白名单校验，审计日志

### 架构理念

**为什么采用 Java + Python 混合架构？**

| 层次       | 技术选型             | 核心理由                                    |
| ---------- | -------------------- | ------------------------------------------- |
| **前端**   | Vue 3 + Element Plus | 企业级 UI 组件库，快速开发                  |
| **业务层** | Java (Spring Boot)   | 企业级稳定性、成熟生态、高并发性能          |
| **AI 层**  | Python (FastAPI)     | 最成熟的 AI 生态（LangChain、Transformers） |
| **数据层** | PostgreSQL           | 强 ACID、丰富索引、JSON 支持                |

**关键设计决策**：

1. ✅ **职责分离**: Java 负责安全和数据，Python 负责 AI 推理
2. ✅ **安全隔离**: Python 不直接访问数据库，SQL 执行由 Java 控制
3. ✅ **性能优化**: Java 使用虚拟线程（JDK 21+），Python 用异步 FastAPI
4. ✅ **可维护性**: 清晰的服务边界，独立部署和扩展

## 🔐 安全设计

### 认证授权

- **JWT Token**: 用户登录后签发 JWT，有效期 24 小时
- **角色权限**: `user` (普通用户), `analyst` (分析师), `admin` (管理员)
- **API 鉴权**: 所有 `/api/v1/*` 接口需验证 JWT

### SQL 安全

```java
// Java Gateway 实现
public class SqlValidator {
    // 1. 白名单校验
    validateTableWhitelist(sql);

    // 2. 黑名单检测
    if (sql.contains("DROP") || sql.contains("DELETE")) {
        throw new SecurityException("Prohibited SQL operation");
    }

    // 3. 只读强制
    executeQuery(sql); // 使用 executeQuery 而非 execute
}
```

### 审计日志

所有操作记录到 `audit_logs` 表：

- 用户登录/登出
- SQL 查询（问题、生成 SQL、执行时间）
- 文档上传
- 配置修改

## 🔄 通信协议

### WebSocket (Java ↔ Python)

**端点**: `ws://localhost:8001/ws`

**消息格式** (JSON-RPC 2.0):

```json
// 请求: Java → Python
{
  "id": "req_abc123",
  "type": "request",
  "method": "nl2sql.generate",
  "params": {
    "question": "本月销售额前10的商品",
    "schema": {...},
    "user_id": 1
  }
}

// 响应: Python → Java
{
  "id": "req_abc123",
  "type": "response",
  "result": {
    "sql": "SELECT ... LIMIT 10",
    "confidence": 0.95,
    "explanation": "..."
  }
}

// 流式响应 (RAG)
{"id": "req_def456", "type": "stream_start"}
{"id": "req_def456", "type": "stream_data", "chunk": {"content": "..."}}
{"id": "req_def456", "type": "stream_end"}
```

**支持的 RPC 方法**:

| 方向          | 方法              | 描述            |
| ------------- | ----------------- | --------------- |
| Java → Python | `nl2sql.generate` | 生成 SQL        |
| Java → Python | `rag.query`       | RAG 检索 (流式) |
| Java → Python | `llm.chat`        | LLM 对话 (流式) |
| Java → Python | `agent.route`     | 意图识别        |
| Python → Java | `db.execute`      | 执行 SQL        |
| Python → Java | `db.get_schema`   | 获取表结构      |
| Python → Java | `cache.get/set`   | 缓存操作        |
