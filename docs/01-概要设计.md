# IDAP 概要设计文档

## 1. 项目概述

**项目名称**: IDAP (Intelligent Data Analysis Platform)
**项目定位**: 企业级智能数据分析平台
**核心价值**: 通过自然语言与数据对话，支持 NL2SQL、RAG 知识检索和智能可视化

---

## 2. 技术架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    前端层 (Port 3000)                        │
│          Vue 3 + Element Plus + Pinia + Vite + Bun          │
└────────────────────────┬────────────────────────────────────┘
                         │ HTTP/SSE
┌────────────────────────▼────────────────────────────────────┐
│                Java Gateway (Port 8080)                      │
│   Spring Boot + Spring Security + JWT + WebSocket           │
│   ┌──────────────┐  ┌──────────────┐  ┌─────────────┐     │
│   │ API Gateway  │  │ SQL Executor │  │ Analytics   │     │
│   │ 认证/授权     │  │ 白名单校验    │  │ 图表生成     │     │

└────────────────────────┬────────────────────────────────────┘
                         │ WebSocket (JSON-RPC)
┌────────────────────────▼────────────────────────────────────┐
│            Python AI Engine (Port 8001)                      │
│     FastAPI + LangChain + LangGraph + WebSocket              │

│   └──────────────┘  └──────────────┘  └─────────────┘     │
└─────────────────────────────────────────────────────────────┘
         │                    │                    │
```

- 用户认证与授权

- 历史查询记录
- 系统配置管理

## 4. 数据模型概览

### 4.1 核心表结构

```
users (用户表)
  ├── user_id (主键)
  ├── username, email, password_hash
  ├── role (user/analyst/admin)
  └── created_at, updated_at

sessions (会话表)
  ├── session_id (主键, UUID)
  ├── user_id (外键)
  ├── title, context (JSONB)
  └── timestamps

queries (查询记录表)
  ├── query_id (主键)
  ├── session_id (外键)
  ├── question, generated_sql, result_data
  ├── chart_config (JSONB)
  └── execution_time_ms, token_count

rag_documents (知识库文档表)
  ├── doc_id (主键)
  ├── title, source_url, content
  └── embedding_model, status

rag_chunks (文档块表)
  ├── chunk_id (主键)
  ├── doc_id (外键)
  ├── chunk_text, embedding (vector)
  └── metadata (JSONB)

products, customers, sales (业务数据表)
```

详见 `02-详细设计-数据库.md`

## 5. 接口设计概览

### 5.1 前端 ↔ Java Gateway

**REST API**

```
POST   /api/v1/auth/login          # 用户登录
GET    /api/v1/auth/me             # 获取用户信息
POST   /api/v1/query               # 提交查询
GET    /api/v1/query/stream        # 流式查询（SSE）
GET    /api/v1/sessions            # 获取会话列表
GET    /api/v1/sessions/:id/history # 获取会话历史
GET    /api/v1/config              # 系统配置
```

### 5.2 Java ↔ Python (WebSocket)

**JSON-RPC 协议**

```json
// 请求
{
  "id": "req_abc123",
  "type": "request",
  "method": "nl2sql.generate",
  "params": { "question": "...", "schema": {...} }
}

// 响应
{
  "id": "req_abc123",
  "type": "response",
  "result": { "sql": "...", "confidence": 0.95 }
}

// 流式推送
{
  "id": "req_def456",
  "type": "stream_data",
  "chunk": { "content": "..." }
}
```

## 7. 性能目标

| 指标            | 目标值 | 说明                 |
| --------------- | ------ | -------------------- |
| NL2SQL 生成延迟 | ≤ 3s   | 从问题到 SQL 生成    |
| SQL 执行时间    | ≤ 5s   | 普通查询执行时间     |
| RAG 检索延迟    | ≤ 2s   | 向量检索 + 答案生成  |
| 端到端延迟      | ≤ 8s   | 用户提问到答案展示   |
| NL2SQL 准确率   | ≥ 75%  | SQL 可执行且结果正确 |
| 并发用户数      | 100+   | 系统支持的并发用户   |
