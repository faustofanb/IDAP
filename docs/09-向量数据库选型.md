# 向量数据库选型方案

## 📊 架构定位

### 数据库职责划分

| 数据库             | 部署位置         | 职责           | 访问方式        |
| ------------------ | ---------------- | -------------- | --------------- |
| **PostgreSQL 16+** | Java Gateway     | 业务数据存储   | Java 直接访问   |
| **Redis 7+**       | Java Gateway     | 缓存、会话存储 | Java 直接访问   |
| **向量数据库**     | Python AI Engine | RAG 向量检索   | Python 独占访问 |

**关键原则**：

- ✅ Java 不访问向量数据库
- ✅ Python 不直接访问 PostgreSQL（通过 Java 的 WebSocket RPC）
- ✅ 向量数据库只服务于 Python AI Engine 的 RAG 模块

---

## 🎯 向量数据库选型

### 方案 1: FAISS（推荐用于开发/MVP）

#### 优势

- ✅ **Meta 开源**，久经考验
- ✅ **本地文件存储**，无需额外部署
- ✅ **内存高效**，单机性能优秀
- ✅ **Python 生态成熟**，LangChain 原生支持
- ✅ **开发迭代快**，无运维负担

#### 劣势

- ❌ 单机方案，无分布式能力
- ❌ 持久化需手动管理（保存/加载索引）
- ❌ 无 Web 管理界面
- ❌ 不支持增量更新（需重建索引）

#### 适用场景

- 🎯 **开发环境**
- 🎯 **MVP 阶段**
- 🎯 **文档量 < 10 万**
- 🎯 **单机部署**

#### 技术栈

```python
pip install faiss-cpu  # CPU 版本
# 或
pip install faiss-gpu  # GPU 版本（需 CUDA）

# LangChain 集成
from langchain.vectorstores import FAISS
from langchain.embeddings import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()
vectorstore = FAISS.from_documents(docs, embeddings)

# 保存索引
vectorstore.save_local("./data/faiss_index")

# 加载索引
vectorstore = FAISS.load_local("./data/faiss_index", embeddings)

# 检索
results = vectorstore.similarity_search("query", k=5)
```

### 方案 2: Milvus（推荐用于生产）

#### 优势

- ✅ **开源向量数据库标准**（CNCF 项目）
- ✅ **分布式架构**，支持水平扩展
- ✅ **持久化存储**，数据安全
- ✅ **丰富的索引算法**（IVF, HNSW, Annoy）
- ✅ **Web 管理界面**（Attu）
- ✅ **支持增量更新**和实时查询
- ✅ **LangChain 原生支持**

#### 劣势

- ❌ 部署复杂（需 etcd, MinIO/S3）
- ❌ 资源占用较大（推荐 4GB+ 内存）
- ❌ 学习曲线较陡

#### 适用场景

- 🎯 **生产环境**
- 🎯 **大规模数据**（> 10 万文档）
- 🎯 **需要高可用**
- 🎯 **多租户场景**

#### 技术栈

```python
pip install pymilvus

# LangChain 集成
from langchain.vectorstores import Milvus
from langchain.embeddings import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()
vectorstore = Milvus(
    embeddings,
    connection_args={"host": "localhost", "port": "19530"},
    collection_name="idap_knowledge"
)

# 插入文档
vectorstore.add_documents(docs)

# 检索
results = vectorstore.similarity_search("query", k=5)
```

### 方案 3: Chroma（备选）

#### 优势

- ✅ **轻量级**，介于 FAISS 和 Milvus 之间
- ✅ **内置持久化**（SQLite/DuckDB）
- ✅ **HTTP API**，支持远程访问
- ✅ **Python 友好**，易于使用

#### 劣势

- ❌ 社区相对较小
- ❌ 大规模数据性能不如 Milvus
- ❌ 企业级功能有限

#### 适用场景

- 🎯 **中小规模应用**
- 🎯 **需要持久化但不想用 Milvus**
- 🎯 **快速原型验证**

#### 技术栈

```python
pip install chromadb

from langchain.vectorstores import Chroma

vectorstore = Chroma(
    persist_directory="./data/chroma",
    embedding_function=embeddings
)
```

## 📋 实施计划

### 第一步：开发环境（使用 FAISS）

```bash
# 1. 安装依赖
cd backend/python-ai-engine
pip install faiss-cpu langchain langchain-openai

# 2. 创建 RAG 服务
# src/services/rag/vector_store.py
```

```python
from langchain.vectorstores import FAISS
from langchain.embeddings import OpenAIEmbeddings
from langchain.text_splitter import RecursiveCharacterTextSplitter
import os

class VectorStoreService:
    def __init__(self, index_path: str = "./data/faiss_index"):
        self.index_path = index_path
        self.embeddings = OpenAIEmbeddings(model="text-embedding-ada-002")
        self.vectorstore = None

        # 如果索引已存在，加载它
        if os.path.exists(index_path):
            self.load_index()

    def load_index(self):
        """加载已有索引"""
        self.vectorstore = FAISS.load_local(
            self.index_path,
            self.embeddings
        )

    def create_index(self, documents: list):
        """创建新索引"""
        text_splitter = RecursiveCharacterTextSplitter(
            chunk_size=1000,
            chunk_overlap=200
        )
        chunks = text_splitter.split_documents(documents)

        self.vectorstore = FAISS.from_documents(chunks, self.embeddings)
        self.save_index()

    def save_index(self):
        """保存索引到磁盘"""
        self.vectorstore.save_local(self.index_path)

    def add_documents(self, documents: list):
        """添加文档（需要重建索引）"""
        if self.vectorstore is None:
            self.create_index(documents)
        else:
            # FAISS 不支持增量更新，需要重建
            existing_docs = []  # 从某处获取已有文档
            all_docs = existing_docs + documents
            self.create_index(all_docs)

    def similarity_search(self, query: str, k: int = 5):
        """相似度检索"""
        if self.vectorstore is None:
            raise ValueError("Vector store not initialized")

        results = self.vectorstore.similarity_search_with_score(
            query,
            k=k
        )
        return results
```

## 🔧 数据库 Schema 调整

### 调整 PostgreSQL 表结构

```sql
-- 1. 删除 rag_chunks 表（不再使用）
DROP TABLE IF EXISTS rag_chunks;

-- 2. 简化 rag_documents 表
ALTER TABLE rag_documents
DROP COLUMN IF EXISTS embedding_model;

-- 3. 添加 vector_store_type 字段
ALTER TABLE rag_documents
ADD COLUMN vector_store_type VARCHAR(20) DEFAULT 'faiss',
ADD COLUMN vector_collection VARCHAR(100);

COMMENT ON COLUMN rag_documents.vector_store_type IS '向量库类型: faiss/milvus/chroma';
COMMENT ON COLUMN rag_documents.vector_collection IS '向量库集合名称';
```

### 新的数据流

```
1. 用户上传文档
   ↓
2. Java Gateway 保存到 rag_documents 表
   ↓
3. Java 通过 WebSocket 通知 Python
   ↓
4. Python AI Engine:
   - 读取文档内容
   - 分块 (RecursiveCharacterTextSplitter)
   - 向量化 (OpenAI Embeddings)
   - 存入 FAISS/Milvus
   ↓
5. 更新 rag_documents 表状态为 'indexed'
```
