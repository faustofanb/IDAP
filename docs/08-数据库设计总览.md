# IDAP 数据库设计文档

## 📊 数据库概览

- **数据库**: PostgreSQL 16+
- **字符集**: UTF-8
- **时区**: UTC
- **总表数**: 12 张表
- **总视图数**: 6 个视图
- **样例数据**: 2600+ 条记录

---

## 🎯 设计原则

1. **规范化**: 符合第三范式（3NF），避免数据冗余
2. **可扩展**: 使用 JSONB 存储灵活数据
3. **性能优化**: 合理索引、分区表（可选）
4. **安全性**: 白名单机制、审计日志
5. **可维护性**: 完整的注释、清晰的命名

---

## 📋 数据表一览

### 核心业务表

#### 1. users - 用户表

**用途**: 存储系统用户信息

| 字段          | 类型         | 说明                     |
| ------------- | ------------ | ------------------------ |
| user_id       | BIGSERIAL    | 主键                     |
| username      | VARCHAR(50)  | 用户名（唯一）           |
| email         | VARCHAR(100) | 邮箱（唯一）             |
| password_hash | VARCHAR(255) | BCrypt 加密密码          |
| role          | VARCHAR(20)  | 角色：user/analyst/admin |
| is_active     | BOOLEAN      | 是否激活                 |

**索引**: username, email, role

**样例数据**: 7 个用户（包含 demo 账号）

---

#### 2. sessions - 会话表

**用途**: 存储用户对话会话

| 字段          | 类型         | 说明             |
| ------------- | ------------ | ---------------- |
| session_id    | UUID         | 主键             |
| user_id       | BIGINT       | 用户 ID（外键）  |
| title         | VARCHAR(200) | 会话标题         |
| context       | JSONB        | 会话上下文元数据 |
| message_count | INTEGER      | 消息数量         |

**索引**: user_id, updated_at, GIN(context)

**样例数据**: 4 个会话

| 字段              | 类型         | 说明                        |

| session_id        | UUID         | 会话 ID（外键）             |
| question          | TEXT         | 用户问题                    |
| intent            | VARCHAR(50)  | 查询意图（nl2sql/rag/chat） |
| generated_sql     | TEXT         | 生成的 SQL                  |

| execution_time_ms | INTEGER      | 执行时间（毫秒）            |
| confidence_score  | DECIMAL(3,2) | AI 置信度                   |

**索引**: session_id, user_id, created_at, intent, GIN(question)

**用途**: NL2SQL 核心表，存储所有查询和结果

## 🎯 NL2SQL 示例查询

### 简单查询

```sql
-- "本月销售额是多少？"
SELECT SUM(final_amount) as total_revenue
FROM sales
WHERE sale_date >= DATE_TRUNC('month', CURRENT_DATE);

-- "销售额TOP10的商品"
SELECT p.product_name, SUM(s.final_amount) as revenue
FROM sales s
JOIN products p ON s.product_id = p.product_id
GROUP BY p.product_name
ORDER BY revenue DESC
LIMIT 10;
```

### 复杂查询

```sql
-- "各地区各渠道的销售对比"
SELECT
    region,
    channel,
    COUNT(*) as order_count,
    SUM(final_amount) as revenue
FROM sales
GROUP BY region, channel
ORDER BY region, revenue DESC;

-- "金卡以上会员的平均消费"
SELECT
    membership_level,
    AVG(total_spent) as avg_spent,
    COUNT(*) as member_count
FROM customers
WHERE membership_level IN ('gold', 'platinum', 'diamond')
GROUP BY membership_level;
```

## 🔐 安全设计

### 1. 白名单机制

- 只允许查询 `table_whitelist` 中的表
- 只能查询 `allowed_columns` 中的列
- 限制最大返回行数

### 2. 审计日志

- 记录所有查询操作
- 包含 SQL、用户、IP、执行时间
- 用于安全审计和故障排查

### 3. 密码安全

- BCrypt 加密（Cost Factor: 10）
