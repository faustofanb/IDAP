## 2. 表结构设计

### 2.1 用户与认证

#### users (用户表)

```sql
CREATE TABLE users (
    user_id BIGSERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    display_name VARCHAR(100),
    role VARCHAR(20) DEFAULT 'user' CHECK (role IN ('user', 'analyst', 'admin')),
    avatar_url VARCHAR(500),
    is_active BOOLEAN DEFAULT true,
    last_login_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_role ON users(role);
```

**字段说明**:

- `password_hash`: BCrypt 加密后的密码
- `role`: 用户角色（user-普通用户, analyst-分析师, admin-管理员）
- `is_active`: 账号是否激活

### 2.2 会话管理

#### sessions (会话表)

```sql
CREATE TABLE sessions (
    session_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id BIGINT NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    title VARCHAR(200) DEFAULT '新对话',
    context JSONB DEFAULT '{}',
    message_count INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_sessions_user_id ON sessions(user_id);
CREATE INDEX idx_sessions_updated_at ON sessions(updated_at DESC);
CREATE INDEX idx_sessions_context ON sessions USING gin(context);
```

**字段说明**:

- `context`: 会话上下文（JSONB），存储元数据
- `message_count`: 消息计数（性能优化）

### 2.3 查询与执行

#### queries (查询记录表)

```sql
CREATE TABLE queries (
    query_id BIGSERIAL PRIMARY KEY,
    session_id UUID REFERENCES sessions(session_id) ON DELETE CASCADE,
    user_id BIGINT NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    question TEXT NOT NULL,
    intent VARCHAR(50),
    generated_sql TEXT,
    executed_sql TEXT,
    sql_status VARCHAR(20) DEFAULT 'pending' CHECK (sql_status IN ('pending', 'success', 'error')),
    result_data JSONB,
    result_rows INTEGER,
    chart_config JSONB,
    answer TEXT,
    error_message TEXT,
    execution_time_ms INTEGER,
    token_count INTEGER,
    model_name VARCHAR(100),
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_queries_session_id ON queries(session_id);
CREATE INDEX idx_queries_user_id ON queries(user_id);
CREATE INDEX idx_queries_created_at ON queries(created_at DESC);
CREATE INDEX idx_queries_sql_status ON queries(sql_status);
CREATE INDEX idx_queries_intent ON queries(intent);
```

**字段说明**:

- `intent`: 查询意图（nl2sql, rag, chat）
- `generated_sql`: AI 生成的原始 SQL
- `executed_sql`: 实际执行的 SQL（可能添加了限制）
- `result_data`: 查询结果（JSON 格式，限制大小）
- `chart_config`: ECharts 图表配置
- `token_count`: LLM Token 消耗

### 2.4 RAG 知识库

#### rag_documents (文档表)

```sql
CREATE TABLE rag_documents (
    doc_id BIGSERIAL PRIMARY KEY,
    title VARCHAR(500) NOT NULL,
    source_type VARCHAR(50) DEFAULT 'manual' CHECK (source_type IN ('file', 'url', 'manual', 'api')),
    source_url TEXT,
    category VARCHAR(100),
    tags TEXT[],
    content TEXT NOT NULL,
    content_hash VARCHAR(64) UNIQUE,
    file_size INTEGER,
    file_type VARCHAR(50),
    embedding_model VARCHAR(100) DEFAULT 'text-embedding-ada-002',
    status VARCHAR(20) DEFAULT 'pending' CHECK (status IN ('pending', 'indexing', 'active', 'error')),
    indexed_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    created_by BIGINT REFERENCES users(user_id)
);

CREATE INDEX idx_rag_documents_category ON rag_documents(category);
CREATE INDEX idx_rag_documents_status ON rag_documents(status);
CREATE INDEX idx_rag_documents_tags ON rag_documents USING gin(tags);
CREATE INDEX idx_rag_documents_content_hash ON rag_documents(content_hash);
```

#### rag_chunks (文档块表)

```sql
CREATE TABLE rag_chunks (
    chunk_id BIGSERIAL PRIMARY KEY,
    doc_id BIGINT NOT NULL REFERENCES rag_documents(doc_id) ON DELETE CASCADE,
    chunk_index INTEGER NOT NULL,
    chunk_text TEXT NOT NULL,
    embedding vector(1536),
    token_count INTEGER,
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(doc_id, chunk_index)
);

CREATE INDEX idx_rag_chunks_doc_id ON rag_chunks(doc_id);
CREATE INDEX idx_rag_chunks_embedding ON rag_chunks USING ivfflat(embedding vector_cosine_ops);
-- 或使用 HNSW: CREATE INDEX idx_rag_chunks_embedding ON rag_chunks USING hnsw(embedding vector_cosine_ops);
```

**字段说明**:

- `embedding`: 文本向量（1536 维，OpenAI ada-002）
- `chunk_index`: 文档内的块序号
- `metadata`: 额外元数据（页码、段落号等）

## 3. 视图设计

### 3.1 销售分析视图

#### daily_sales_summary (每日销售汇总)

```sql
CREATE VIEW daily_sales_summary AS
SELECT
    sale_date,
    COUNT(*) as order_count,
    COUNT(DISTINCT customer_id) as customer_count,
    SUM(total_amount) as total_revenue,
    AVG(total_amount) as avg_order_value,
    SUM(quantity) as total_quantity
FROM sales
GROUP BY sale_date
ORDER BY sale_date DESC;
```

#### product_sales_ranking (产品销售排名)

```sql
CREATE VIEW product_sales_ranking AS
SELECT
    p.product_id,
    p.product_name,
    p.category,
    p.brand,
    COUNT(DISTINCT s.order_id) as order_count,
    SUM(s.quantity) as total_quantity_sold,
    SUM(s.total_amount) as total_revenue,
    AVG(s.total_amount) as avg_order_value
FROM products p
LEFT JOIN sales s ON p.product_id = s.product_id
GROUP BY p.product_id, p.product_name, p.category, p.brand
ORDER BY total_revenue DESC;
```

## 4. 触发器

### 4.1 自动更新时间戳

```sql
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_users_updated_at BEFORE UPDATE ON users
    FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_sessions_updated_at BEFORE UPDATE ON sessions
    FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

## 6. 性能优化

### 6.1 索引策略

- **B-Tree 索引**: 主键、外键、常用查询字段
- **GIN 索引**: JSONB 字段、数组字段、全文搜索
- **GiST/HNSW 索引**: 向量相似度搜索

### 6.2 分区策略

- `sales` 表按 `sale_date` 月度分区（Range Partitioning）
- `audit_logs` 表按 `created_at` 月度分区

### 6.3 查询优化

- 使用 Prepared Statement
- 限制返回行数（默认 1000 行）
- 查询超时设置（30 秒）
- 使用连接池（HikariCP，最大 20 连接）

### 7.1 备份策略

- **增量备份**: 每 6 小时
