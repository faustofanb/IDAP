## 1. 启动数据库

### 方式 1: 使用 Docker Compose（推荐）

```bash
cd scripts/docker
./01-docker-manager.sh start-dev
```

**预期输出**:

```
✅ PostgreSQL 启动成功
✅ Redis 启动成功
✅ 执行数据库初始化脚本...
✅ 数据统计: 12表, 6视图, 2600+记录
```

### 方式 2: Docker 命令直接启动

```bash
docker run -d \
  --name idap-postgres \
  -p 5432:5432 \
  -e POSTGRES_DB=idap \
  -e POSTGRES_USER=idap \
  -e POSTGRES_PASSWORD=idap123 \
  postgres:16
```

### 验证启动

```bash
# 检查容器状态
docker ps | grep postgres

# 查看日志
docker logs idap-postgres

# 测试连接
docker exec idap-postgres psql -U idap -d idap -c "SELECT version();"
```

## 2. 使用 MCP 操作数据库

### 2.1 配置 PostgreSQL MCP

**文件位置**: `~/Library/Application Support/Code/User/mcp.json`

```json
{
  "servers": {
    "postgres": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-postgres",
        "postgresql://idap:idap123@localhost:5432/idap"
      ]
    }
  }
}
```

### 2.2 使用 MCP 查询

在 VS Code 中，通过 Copilot 聊天窗口：

```
@workspace 使用 MCP 查询所有表
```

或直接在代码中：

```typescript
// 示例：使用 MCP 工具
const result = await mcp_postgres_query({
  sql: "SELECT * FROM users LIMIT 5;",
});
```

### 2.3 MCP 常见问题

#### 问题 1: MCP 连接失败

**原因**: Docker 容器未启动或连接字符串错误

**解决**:

```bash
# 1. 确保数据库运行
docker ps | grep postgres

# 2. 测试连接
psql postgresql://idap:idap123@localhost:5432/idap -c "SELECT 1;"

# 3. 检查 MCP 配置
cat ~/Library/Application\ Support/Code/User/mcp.json
```

#### 问题 2: MCP 权限错误

**原因**: npx 未安装或权限不足

**解决**:

```bash
# 安装 Node.js (如果未安装)
brew install node

# 测试 npx
npx -y @modelcontextprotocol/server-postgres --version

# 手动测试 MCP 连接
npx -y @modelcontextprotocol/server-postgres \
  postgresql://idap:idap123@localhost:5432/idap
```

#### 问题 3: MCP 响应超时

**原因**: 查询时间过长

**解决**:

- 使用 `LIMIT` 限制返回行数
- 优化查询，添加索引
- 增加超时时间

## 3. 使用 Navicat 操作

### 3.1 创建连接

1. 打开 Navicat Premium
2. 点击 **连接** → **PostgreSQL**
3. 填写连接信息：
   - **连接名**: IDAP Local
   - **主机**: localhost
   - **端口**: 5432
   - **数据库**: idap
   - **用户名**: idap
   - **密码**: idap123
4. 点击 **测试连接**
5. 点击 **确定**

### 3.2 查看数据

1. 展开连接 → `idap` → `public` → `表`
2. 双击表名查看数据
3. 右键 → **设计表** 查看结构

### 3.3 执行 SQL

1. 右键数据库 → **新建查询**
2. 输入 SQL 语句
3. 点击 **运行** 或按 `Ctrl/Cmd + R`

### 3.4 导出数据

1. 右键表 → **转储 SQL 文件**
2. 选择导出选项
3. 保存文件

## 5. 常用查询示例

### 5.1 系统信息

```sql
-- 数据库版本
SELECT version();

-- 当前用户
SELECT current_user;

-- 当前数据库
SELECT current_database();

-- 连接信息
SELECT * FROM pg_stat_activity WHERE datname = 'idap';

-- 数据库大小
SELECT pg_size_pretty(pg_database_size('idap'));
```

### 5.2 表和索引

```sql
-- 所有表及大小
SELECT
    schemaname,
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

-- 表记录数统计
SELECT
    schemaname,
    tablename,
    n_tup_ins - n_tup_del AS row_count
FROM pg_stat_user_tables
ORDER BY n_tup_ins - n_tup_del DESC;

-- 索引使用情况
SELECT
    schemaname,
    tablename,
    indexname,
    idx_scan,
    idx_tup_read,
    idx_tup_fetch
FROM pg_stat_user_indexes
ORDER BY idx_scan DESC;

-- 未使用的索引
SELECT
    schemaname,
    tablename,
    indexname,
    idx_scan
FROM pg_stat_user_indexes
WHERE idx_scan = 0
  AND indexname NOT LIKE '%_pkey';
```

### 5.3 业务查询

```sql
-- 用户列表
SELECT user_id, username, email, role, is_active, last_login_at
FROM users
ORDER BY user_id;

-- 产品分类统计
SELECT
    category,
    COUNT(*) as product_count,
    AVG(price) as avg_price,
    MIN(price) as min_price,
    MAX(price) as max_price
FROM products
GROUP BY category
ORDER BY product_count DESC;

-- 每日销售汇总（最近7天）
SELECT * FROM daily_sales_summary
ORDER BY sale_date DESC
LIMIT 7;

-- TOP 10 畅销商品
SELECT * FROM product_sales_ranking
LIMIT 10;

-- 客户地区分布
SELECT
    region,
    COUNT(*) as customer_count,
    SUM(total_spent) as total_revenue,
    AVG(total_spent) as avg_spent
FROM customers
GROUP BY region
ORDER BY total_revenue DESC;

-- RAG 文档列表
SELECT
    doc_id,
    title,
    category,
    status,
    vector_store_type,
    LENGTH(content) as content_length,
    created_at
FROM rag_documents
ORDER BY created_at DESC;
```

### 5.4 性能分析

```sql
-- 慢查询统计
SELECT
    query,
    calls,
    total_time,
    mean_time,
    max_time
FROM pg_stat_statements
ORDER BY mean_time DESC
LIMIT 10;

-- 表膨胀检查
SELECT
    schemaname,
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS total_size,
    n_live_tup,
    n_dead_tup,
    ROUND(100 * n_dead_tup / NULLIF(n_live_tup + n_dead_tup, 0), 2) AS dead_ratio
FROM pg_stat_user_tables
WHERE n_live_tup > 0
ORDER BY n_dead_tup DESC;

-- 锁等待
SELECT
    pid,
    usename,
    pg_blocking_pids(pid) as blocked_by,
    query as blocked_query
FROM pg_stat_activity
WHERE cardinality(pg_blocking_pids(pid)) > 0;
```

## 6. 故障排查

### 6.1 连接失败

**症状**: 无法连接到数据库

**排查步骤**:

```bash
# 1. 检查容器是否运行
docker ps | grep postgres

# 2. 检查端口是否监听
lsof -i :5432
netstat -an | grep 5432

# 3. 检查容器日志
docker logs idap-postgres | tail -50

# 4. 尝试容器内连接
docker exec -it idap-postgres psql -U idap -d idap

# 5. 检查防火墙
sudo ufw status
```

**解决方案**:

```bash
# 重启容器
docker restart idap-postgres

# 完全重建
cd scripts/docker
./01-docker-manager.sh stop-dev
./01-docker-manager.sh start-dev
```

### 6.2 数据未初始化

**症状**: 表不存在或数据为空

**排查步骤**:

```bash
# 1. 检查表是否存在
docker exec idap-postgres psql -U idap -d idap -c "\dt"

# 2. 检查初始化脚本是否执行
docker exec idap-postgres ls -la /docker-entrypoint-initdb.d/

# 3. 查看容器创建时间
docker inspect idap-postgres | grep Created
```

**解决方案**:

```bash
# 方式 1: 手动执行初始化脚本
docker exec -i idap-postgres bash < database/init-db.sh

# 方式 2: 重建容器和数据卷
cd scripts/docker
./01-docker-manager.sh stop-dev
docker volume rm idap_postgres_data
./01-docker-manager.sh start-dev
```

### 6.3 性能问题

**症状**: 查询慢、超时

**排查步骤**:

```sql
-- 检查慢查询
SELECT query, mean_time, calls
FROM pg_stat_statements
ORDER BY mean_time DESC
LIMIT 10;

-- 检查锁
SELECT * FROM pg_locks WHERE NOT granted;

-- 检查连接数
SELECT count(*) FROM pg_stat_activity;
```

**解决方案**:

```sql
-- 分析表
ANALYZE users;
ANALYZE products;
ANALYZE sales;

-- 重建索引
REINDEX TABLE sales;

-- 清理死元组
VACUUM ANALYZE sales;

-- 杀死慢查询
SELECT pg_terminate_backend(pid)
FROM pg_stat_activity
WHERE state = 'active' AND query_start < now() - interval '5 minutes';
```

### 6.4 权限问题

**症状**: 权限拒绝错误

**排查步骤**:

```sql
-- 检查用户权限
SELECT * FROM pg_user WHERE usename = 'idap';

-- 检查表权限
SELECT grantee, privilege_type
FROM information_schema.table_privileges
WHERE table_name = 'users';
```

**解决方案**:

```sql
-- 授予权限
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO idap;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO idap;

-- 设置默认权限
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO idap;
```

## 7. 维护操作

### 7.1 备份数据库

```bash
# 完整备份
docker exec idap-postgres pg_dump -U idap idap > backup_$(date +%Y%m%d).sql

# 只备份结构
docker exec idap-postgres pg_dump -U idap --schema-only idap > schema.sql

# 只备份数据
docker exec idap-postgres pg_dump -U idap --data-only idap > data.sql

# 压缩备份
docker exec idap-postgres pg_dump -U idap idap | gzip > backup.sql.gz
```

### 7.2 恢复数据库

```bash
# 恢复备份
docker exec -i idap-postgres psql -U idap -d idap < backup.sql

# 解压并恢复
gunzip -c backup.sql.gz | docker exec -i idap-postgres psql -U idap -d idap
```

### 7.3 数据清理

```sql
-- 清理旧日志
DELETE FROM audit_logs WHERE created_at < NOW() - INTERVAL '90 days';

-- 清理测试数据
DELETE FROM queries WHERE user_id IN (SELECT user_id FROM users WHERE username LIKE 'test%');

-- 清理会话
DELETE FROM sessions WHERE updated_at < NOW() - INTERVAL '30 days';

-- 清理死元组
VACUUM FULL;
```

### 7.4 性能优化

```sql
-- 更新统计信息
ANALYZE;

-- 重建索引
REINDEX DATABASE idap;

-- 清理并分析
VACUUM ANALYZE;

-- 检查表膨胀
SELECT
    schemaname || '.' || tablename AS table,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size,
    n_dead_tup
FROM pg_stat_user_tables
WHERE n_dead_tup > 1000
ORDER BY n_dead_tup DESC;
```

## 8. 最佳实践

### 8.1 开发环境

- ✅ 使用 Docker Compose 管理数据库
- ✅ 定期备份开发数据
- ✅ 使用迁移脚本管理 Schema 变更
- ✅ 避免在生产环境运行测试 SQL

### 8.2 查询优化

- ✅ 始终使用 `LIMIT` 限制返回行数
- ✅ 使用 `EXPLAIN ANALYZE` 分析查询计划
- ✅ 避免 `SELECT *`，只查询需要的列
- ✅ 合理使用索引

### 8.3 安全实践

- ✅ 不在代码中硬编码密码
- ✅ 使用环境变量存储敏感信息
- ✅ 定期更新密码
- ✅ 限制数据库访问权限

### 8.4 监控维护

- ✅ 定期检查日志
- ✅ 监控慢查询
- ✅ 定期备份
- ✅ 定期清理旧数据
