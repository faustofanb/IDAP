# IDAP 架构调整说明 - 向量数据库分离

## 📅 更新日期

2025-01-15

## 🎯 调整目标

将向量存储从 PostgreSQL 迁移到 Python AI Engine 侧，采用专业的向量数据库（FAISS/Milvus）。

---

## 🔄 调整前后对比

### ❌ 调整前（错误设计）

```
PostgreSQL 16+
├── 业务数据表 (users, products, sales...)
├── RAG 文档表 (rag_documents)
└── RAG 向量表 (rag_chunks)  ❌ 问题所在
    ├── chunk_text
    ├── vector_id (指向 Milvus)
    └── embedding (如果用 pgvector)

问题:
1. Java 不需要访问向量数据
2. PostgreSQL 不适合做向量检索（即使有 pgvector）
3. rag_chunks 表设计混乱（既有 vector_id 又有 embedding）
4. 跨服务数据同步复杂
```

### ✅ 调整后（正确设计）

```
┌─────────────────────────────────────────────────┐
│       Java Gateway (Port 8080)                  │
├─────────────────────────────────────────────────┤
│  PostgreSQL 16+                                 │
│  ├── users, sessions, queries                   │
│  ├── products, customers, sales                 │
│  ├── rag_documents (文档元数据)                  │
│  │   ├── title, content, status                 │
│  │   ├── vector_store_type (faiss/milvus)       │
│  │   └── vector_collection (集合名)             │
│  └── system_config, audit_logs                  │
│                                                  │
│  Redis 7+                                        │
│  └── 缓存、会话                                  │
└─────────────────────────────────────────────────┘
         │
         │ WebSocket (JSON-RPC 2.0)
         │
┌────────▼────────────────────────────────────────┐
│    Python AI Engine (Port 8001)                 │
├─────────────────────────────────────────────────┤
│  向量存储 (独占访问)                             │
│                                                  │
│  开发环境: FAISS                                 │
│  ├── ./data/faiss_index/                        │
│  ├── index.faiss (向量索引)                      │
│  └── index.pkl (元数据)                          │
│                                                  │
│  生产环境: Milvus 2.3+                           │
│  ├── Collection: idap_knowledge                 │
│  ├── Vectors: embeddings (1536维)               │
│  └── Metadata: doc_id, chunk_index, text        │
└─────────────────────────────────────────────────┘

优势:
1. ✅ 职责清晰：Java 管业务数据，Python 管向量检索
2. ✅ 性能最优：FAISS/Milvus 专为向量检索设计
3. ✅ 易于维护：向量索引与业务数据解耦
4. ✅ 灵活切换：可轻松切换向量数据库类型
```

## 📊 数据库变更

### PostgreSQL Schema 调整

#### 1. 删除 `rag_chunks` 表

```sql
-- 不再需要，向量块由 FAISS/Milvus 管理
DROP TABLE IF EXISTS rag_chunks CASCADE;
```

#### 2. 调整 `rag_documents` 表

```sql
-- 删除字段
ALTER TABLE rag_documents
DROP COLUMN IF EXISTS embedding_model;  -- 不再需要

-- 新增字段
ALTER TABLE rag_documents
ADD COLUMN vector_store_type VARCHAR(20) DEFAULT 'faiss',  -- faiss/milvus/chroma
ADD COLUMN vector_collection VARCHAR(100),                 -- 集合名称
ADD COLUMN chunk_count INTEGER DEFAULT 0,                  -- 分块数量
ADD COLUMN error_message TEXT;                             -- 错误信息
```

#### 3. 新的表结构

```sql
rag_documents (
    doc_id              BIGSERIAL PRIMARY KEY,
    title               VARCHAR(500),
    content             TEXT,
    content_hash        VARCHAR(64) UNIQUE,
    source_type         VARCHAR(50),
    category            VARCHAR(100),
    tags                TEXT[],
    -- 向量存储元数据
    vector_store_type   VARCHAR(20),      -- 使用哪个向量库
    vector_collection   VARCHAR(100),     -- 集合/索引名称
    chunk_count         INTEGER,          -- 分块数量
    -- 状态管理
    status              VARCHAR(20),      -- pending/indexing/active/error
    error_message       TEXT,             -- 错误详情
    indexed_at          TIMESTAMPTZ,
    created_at          TIMESTAMPTZ,
    updated_at          TIMESTAMPTZ,
    created_by          BIGINT
)
```

## 🔄 数据流调整

### 文档上传与向量化流程

```
1. 用户上传文档
   ↓
2. Java Gateway
   ├── 保存文档到 rag_documents 表
   ├── status = 'pending'
   └── 生成 doc_id
   ↓
3. Java 通过 WebSocket 通知 Python
   {
       "method": "rag.index_document",
       "params": {
           "doc_id": 123,
           "title": "产品手册",
           "content": "文档内容...",
           "vector_store": "faiss"  // or "milvus"
       }
   }
   ↓
4. Python AI Engine
   ├── 文档分块 (RecursiveCharacterTextSplitter)
   │   └── chunk_size=1000, overlap=200
   ├── 向量化 (OpenAI Embeddings)
   │   └── model: text-embedding-ada-002
   ├── 存入向量库
   │   ├── FAISS: 保存到 ./data/faiss_index/
   │   └── Milvus: 插入到 collection
   └── 返回结果
   ↓
5. Java Gateway 更新数据库
   ├── status = 'active'
   ├── vector_store_type = 'faiss'
   ├── vector_collection = 'idap_knowledge_123'
   ├── chunk_count = 25
   └── indexed_at = CURRENT_TIMESTAMP
```

### RAG 查询流程

```
1. 用户提问："如何使用数据分析功能？"
   ↓
2. Java Gateway 转发到 Python
   {
       "method": "rag.query",
       "params": {
           "question": "如何使用数据分析功能？",
           "top_k": 5,
           "vector_store": "faiss"
       }
   }
   ↓
3. Python AI Engine
   ├── 问题向量化
   ├── 向量检索 (FAISS/Milvus)
   │   └── similarity_search(query_vector, k=5)
   ├── 获取 Top-K 文档块
   ├── 重排序 (可选)
   └── LLM 生成答案
       ├── Prompt: 问题 + 检索到的文档
       └── 返回答案 + 引用来源
   ↓
4. Java Gateway 流式返回前端
   ├── 答案内容 (SSE)
   └── 引用来源 [{doc_id, title, similarity}]
```

## 🛠️ 技术实现

### Python RAG 服务接口

```python
# src/services/rag/vector_store.py

from typing import Literal

VectorStoreType = Literal["faiss", "milvus", "chroma"]

class VectorStoreFactory:
    @staticmethod
    def create(store_type: VectorStoreType):
        if store_type == "faiss":
            return FAISSVectorStore()
        elif store_type == "milvus":
            return MilvusVectorStore()
        elif store_type == "chroma":
            return ChromaVectorStore()
        else:
            raise ValueError(f"Unsupported vector store: {store_type}")

class VectorStoreService:
    def __init__(self, store_type: VectorStoreType = "faiss"):
        self.store = VectorStoreFactory.create(store_type)

    def index_document(self, doc_id: int, content: str, metadata: dict):
        """索引文档"""
        # 1. 分块
        chunks = self.text_splitter.split_text(content)

        # 2. 向量化
        embeddings = self.embeddings.embed_documents(chunks)

        # 3. 存入向量库
        collection_name = f"idap_knowledge_{doc_id}"
        self.store.add_documents(
            texts=chunks,
            embeddings=embeddings,
            metadatas=[{**metadata, "chunk_index": i} for i in range(len(chunks))],
            collection_name=collection_name
        )

        return {
            "collection_name": collection_name,
            "chunk_count": len(chunks)
        }

    def query(self, question: str, top_k: int = 5):
        """RAG 检索"""
        results = self.store.similarity_search_with_score(question, k=top_k)
        return results
```

### Java WebSocket RPC 调用

```java
// Java Gateway 调用 Python
public CompletableFuture<IndexResult> indexDocument(Long docId, String content) {
    JsonObject request = new JsonObject();
    request.addProperty("id", UUID.randomUUID().toString());
    request.addProperty("method", "rag.index_document");

    JsonObject params = new JsonObject();
    params.addProperty("doc_id", docId);
    params.addProperty("content", content);
    params.addProperty("vector_store", "faiss");  // 从配置读取
    request.add("params", params);

    return pythonAIClient.sendRequest(request);
}
```
