# 002-智能问数：计划（Plan）

## 目标

- 为多租户进销存场景提供“自然语言 → 结构化请求 → 受控执行 → 表格/图表/洞察”的最小可行能力（MVP）。
- 全流程可审计、可回放；无后端时前端可用 Mock 独立演示。

## 范围

- 前端：会话式问数 UI、表格/图表/洞察三种结果展示、WebSocket 流展示。
- 后端：WebSocket 会话接口、请求校验（JWT/RBAC/数据权限）、查询执行器（MVP 可模板/内存）。
- AI 服务：仅输出 RequestEnvelope，不接 DB，不拼接 SQL；解析失败回退模板问答。
- 契约：RequestEnvelope/InsightPayload 等 JSON Schema，生成多语言类型（TS/Java/Python）。

## 非目标（本阶段不做）

- 跨租/跨机构运营能力。
- 实时链路 Kafka/异步补偿。
- 复杂可视化编辑器。

## 里程碑

1. M1 前端 Mock 问数流（结果三合一）
2. M2 后端 WS MVP + 鉴权/数据权限钩子（伪实现）
3. M3 AI IntentGraph 最小链路 + 回退策略
4. M4 契约与类型生成流水线（CI 内）
5. M5 审计/回放与报表订阅基础

## 风险

- 多租户数据权限复杂度高，需尽早抽象策略模型。
- LLM 解析不稳定，需回退与提示工程兜底。
- 契约漂移风险，需在 CI 中强制校验。

## 验收

- Demo：同一问题在 Mock / MVP 后端均可跑通。
- 安全：越权用例被显式阻断并审计。
- 质量：关键路径 95p < 2s（Mock 环境），失败回退可用。
