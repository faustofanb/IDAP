# 001-进销存业务系统：实施计划（Plan）

## 目标（当前优先）

- 先完成“进销存后台（采购-销售-库存）”最小可用版本（MVP），包含租户隔离、RBAC/数据权限、审计与基础报表。
- 智能问数（AI）放到后续阶段（功能 002）在业务系统之上扩展。

## 范围

- 采购：供应商、采购单（创建/审核/入库/退货）、应付台账。
- 销售：客户、销售单（创建/审核/出库/退货）、应收台账。
- 库存：入库/出库/调拨/盘点，批次/序列号管理（MVP 可简化），库存快照。
- 权限与多租户：JWT 鉴权、RBAC（URL/资源粒度）、数据权限（按租户/机构/仓库范围），所有请求显式携带 tenantId。
- 报表：基础统计（按日/周/月），导出与订阅（可后置为异步）。

## 非目标（当前阶段不纳入）

- 智能问数/AI 能力（转至 002）。
- 复杂 WMS（波次/策略/路径优化）、复杂结算/财务对接。
- 实时消息总线与大规模数据仓库对接。

## 里程碑

1. M1 多租户与权限基线：租户/机构/用户/角色/资源模型 + JWT 贯通
2. M2 库存最小闭环：商品、仓库、入库/出库/库存台账 API + 审计
3. M3 采购闭环：供应商、采购单流程 + 退货
4. M4 销售闭环：客户、销售单流程 + 退货
5. M5 报表与导出：基础 KPI（库存余额、周转、采销额）与导出
6. M6 观测与压测：日志/追踪/指标、p95 指标达标

## 风险与缓解

- 数据权限复杂：提前固化按租户/机构/仓库的范围策略；在仓储层实现全局过滤。
- 审批流差异：采购/销售可先采用简化状态机，后续再细化审批角色与回退路径。
- 台账一致性：MVP 阶段以内存/单库事务为主，避免分布式事务；提供对账脚本。

## 验收

- 采购/销售/库存三条主流程能在多租户隔离下完成端到端操作；审计与权限生效。
- 报表和导出可用；关键路径 p95 < 2s（MVP 规模）。
- 文档与任务同步更新，PR 引用对应条目。
