# 000-项目实施计划（Program Plan）

## 愿景与目标

- 构建面向多租户的进销存后台 + 智能问数平台，提供“自然语言 → 结构化请求 → 受控执行 → 报表/洞察”的体验。
- 以契约驱动、可观测与安全合规为护栏，逐步从 Mock 演进到真实数据链路。

## 范围（项目级）

- 前端：问数 UI、表/图/洞察组件、WebSocket 流式展示、Mock 支持。
- 后端：WS 会话接口、JWT/RBAC/数据权限、查询执行器（模板/引擎）、审计与回放。
- AI 服务：仅产出 RequestEnvelope，具备回退策略与遥测。
- 契约与生成：JSON Schema → TS/Java/Python 类型生成（CI）。
- 可观测与安全：日志/追踪/指标统一，审计闭环与越权阻断。

## 非目标（当前阶段不纳入）

- 全量实时链路与大规模数据仓库治理。
- 自定义图表编辑器与复杂报表设计器。

## 里程碑（项目横向）

1. P0 基线搭建：目录结构、规约落地、Mock 可演示
2. M1 FE Mock 问数流：表/图/洞察三合一
3. M2 BE WS MVP：握手校验、tenant 注入、伪执行器
4. M3 AI IntentGraph 最小链路 + 回退策略
5. M4 契约与类型生成流水线（CI）
6. M5 安全/观测/审计最小闭环
7. M6 真实数据集成与性能基准

## 风险与缓解

- 契约漂移：在 CI 中强校验 + 版本化；破坏性变更要求迁移说明。
- 数据权限复杂：尽早抽象行/列/范围策略与租户隔离钩子。
- LLM 不稳定：提示工程 + 模板化回退，关键路径无阻塞。

## 验收标准（项目级）

- Demo：同一问题在 Mock 与 MVP 后端均可跑通。
- 安全：越权用例被阻断并审计留痕。
- 质量：Mock 环境关键路径 p95 < 2s；超时自动降级。
- 文档：plan/spec/tasks 与 contracts 同步更新，可供新成员 1 分钟上手。
