## 四、技术债务与风险

### 4.1 当前技术债务

- ⚠️ 前端页面为占位页面，需要实现具体业务逻辑
- ⚠️ 缺少单元测试与集成测试
- ⚠️ 缺少 API Mock 数据（需要 Mock Service Worker 或 JSON Server）

### 4.2 潜在风险

| 风险                 | 影响 | 缓解措施                            |
| -------------------- | ---- | ----------------------------------- |
| 多租户数据隔离不完善 | 高   | 强制在 ORM 层添加租户 ID 全局过滤器 |
| 权限校验遗漏         | 高   | 使用 Spring Security + 自定义注解   |
| 性能瓶颈             | 中   | 引入 Redis 缓存 + 数据库索引优化    |
| 前后端接口不一致     | 中   | 使用 OpenAPI 生成器自动生成类型定义 |

### 4.3 待决策事项

- [ ] 数据库选型：PostgreSQL vs MySQL（建议 PostgreSQL，支持 JSONB）
- [ ] 缓存策略：Redis vs Caffeine（建议 Redis，支持分布式）
- [ ] 消息队列：Kafka vs RabbitMQ（建议 Kafka，项目已规划）
- [ ] 部署方式：Docker Compose vs Kubernetes（MVP 建议 Docker Compose）
